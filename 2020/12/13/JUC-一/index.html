JUC

之前都是在并发条件下线程不安全的集合，因此 我们就需要使用到锁以及一些保护措施来控制线程的执行， JDK java.util.concurrent 包（JUC）中 为我们提供了丰富的并发控制组件，来满足我们的需求

多线程

多线程下如何保证资源的安全性？

// 资源
class Ticket{
    private int count = 50;
    public void sell(){
        if(count > 0) {
            System.out.println(Thread.currentThread().getName() + "卖了第" + (count--) + "张票");
        }
    }
}

// 多线程操作同一资源
Ticket ticket = new Ticket();
new Thread(() -> {ticket.sell();},"线程0").start();
new Thread(() -> {ticket.sell();},"线程1").start();
new Thread(() -> {ticket.sell();},"线程2").start();
# 结果
线程1卖了第25张票
线程1卖了第24张票
线程0卖了第41张票
线程0卖了第23张票
线程0卖了第22张票

可以看到该结果是不安全的，因此需要加锁

锁
synchronized
public synchronized void sell(){
        if(count > 0) {
            System.out.println(Thread.currentThread().getName() + "卖了第" + (count--) + "张票");
        }
    }
Lock
public void sell(){
        Lock lock = new ReentrantLock();
        // 上锁
        lock.lock();
        try {
            if(count > 0) {
                System.out.println(Thread.currentThread().getName() + "卖了第" + (count--) + "张票");
            }
        } finally {
            lock.unlock();
        }
    }

两者之间的区别

  1. Synchronized是关键字，内置语言实现，Lock是接口
  2. Synchronized在线程发生异常时会自动释放锁，因此不会发生异常死锁。Lock异常时不会自动释放锁，所以需要在finally中实现释放锁。
  3. Lock是可以中断锁，Synchronized是非中断锁，必须等待线程执行完成释放锁。

// synchronized
class Product{
    private int num = 0;
    public synchronized void produce() {
        if(num > 0){
            // 等待
               this.wait();
        }
        // 生产产品
        System.out.println(Thread.currentThread().getName() + " => " + (++num);
        // 唤醒
        this.notifyAll();
    }

    public synchronized void consumer() {
        if(num <= 0){
            this.wait();
        }
        System.out.println(Thread.currentThread().getName() + " => " + (--num);
        this.notifyAll();
    }
}

class Product{
    private int num = 0;
    public synchronized void produce() {
        while(num > 0){
            // 等待
            try {
                this.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        // 生产产品
        num++;
        System.out.println(Thread.currentThread().getName() + " => " + num);
        // 唤醒
        this.notifyAll();
    }

    public synchronized void consumer() {
        while(num <= 0){
            try {
                this.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        num--;
        System.out.println(Thread.currentThread().getName() + " => " + num);
        this.notifyAll();
    }
}

class Product2{

    private int num = 0;
    private Lock lock = new ReentrantLock();
    private Condition condition = lock.newCondition();

    public void produce() {
        lock.lock();
        try {
            while (num > 0) {
                // 等待
                condition.await();
            }
            // 生产产品
            System.out.println(Thread.currentThread().getName() + " => " + (++num));
            // 唤醒
            condition.signalAll();
        }catch (Exception e){

        } finally {
            lock.unlock();
        }
    }

    public void consumer() {
        lock.lock();
        try {
            while (num <= 0) {
                condition.await();
            }
            System.out.println(Thread.currentThread().getName() + " => " + (--num));
            condition.signalAll();
        }catch (Exception e){

        } finally {
            lock.unlock();
        }
    }

CountDownLatch

CountDownLatch是一个同步工具类，用来协调多个线程之间的同步，或者说起到线程之间的通信（而不是用作互斥的作用） 

// 声明
CountDownLatch countDownLatch = new CountDownLatch(2);

// 创建两个线程
for(int i = 1; i < 2; i++) {
    final int temp = i;
    new Thread(() -> {
        System.out.println(temp+"线程执行开始");
        try {
            TimeUnit.SECONDS.sleep(3);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println(temp+"线程执行结束");
        countDownLatch.countDown();
    },String.valueOf(temp)).start();
}

// 另外一个线程
System.out.println("主线程等待。。。。");
countDownLatch.await(); // await就会等待countDownLatch为零的时候
System.out.println("主线程结束等待。。。。");

用法

某一线程在开始运行前等待n个线程执行完毕。将CountDownLatch的计数器初始化为n，每当一个任务线程执行 countdownLatch.countDown()完毕，就将计数器减1，当计数器的值变为0时，在CountDownLatch上await()的线程就会被唤醒。一个典型应用场景就是启动一个服务时，主线程需要等待多个组件加载完毕，之后再继续执行
